Linux 系统常用命令

sh、bash、source

sh、bash和source是Linux和类Unix系统中常见的命令，它们在Shell脚本和命令行操作中扮演着不同的角色：

sh

  • sh 是Unix系统上的原始shell程序，由Steve Bourne编写。它是最早的Shell脚本语言之一，提供了基本的脚步执行功能。随着时间的推移，出现了许多基于sh的扩展和改进版本，如bash、zsh、ksh等。
  • 在许多系统中，sh已经被更现代的Shell如bash代替，但任然被保留为指向系统默认Shell的链接，或者用于兼容性目的。
  • 使用sh运行脚本通常意味着脚本将在一个新的shell进程中执行，而不是当前的shell环境中。例如，sh script.sh会启动 一个新的sh进程来运行script.sh。

bash

  • bash:是sh的一个自由软件替代版本，由GNU项目开发。bash兼容sh（意味着他可以运行sh脚本），但添加了许多其他功能，如命令行补全，命令历史、成学会完成等，使其成为许多Linux发行版和MacOS的默认shell。
  • 使用bash运行脚本会在bash环境中执行该脚本，可以访问bash的索引扩展功能和改进。例如，bash script.sh会使用bash来运行script.sh。

source

  • source 是一个shell命令，用于当前shell环境中读取并执行指定的Shell脚本文件。这意味着脚本中定义的任何变量或函数都将在当前shell会话中可用，而不是像使用sh或bash命令那样在新的shell进程中执行脚本。
  • source命令使得脚本对当前shell环境产生影响，常用于配置环境变量或定义函数等场景。
  • 在bash中，.(点命令)时source的同义词，即source script.sh和. script.sh在bash中作用相同。

总结来说，sh和bash都可以用来启动新的shell进程执行脚本，其中bash提供了比sh更丰富的特性和改进。source或（.）命令用于在当前shell环境中执行脚本，使脚本中的变量或函数更改对当前会话有效。

用户、用户组

在Ubuntu中，可以使用useradd命令来创建新的用户并将其添加到特定的用户组。 使用命令：

sudo useradd -m -s /bin/bash -g <group_name> <username>
  • -m：自动为该用户创建主目录（如果不存在）
  • -s /bin/bash：设置默认登录Shell为Bash
  • <group_name>：要将用途添加到的用户组名称
  • <username>：要创建的新用户的用户名 注意：上述命令需要以管理员身份运行（通过sudo或者root用户）才能执行
